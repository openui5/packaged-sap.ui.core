Manual Integration Test: Create and Change Sales Orders via Auto/Direct application groups

DESCRIPTION

Test whether POST and PATCH request for application groups with SubmitMode.Auto/.Direct are parked and restarted automatically

PREPARATION

To check console errors open the developer tools before starting the application and check with each step, that there are only expected errors and warnings.

Testing only possible with realOData=true

Start the Sales Orders application with following URL (adjust host and port if necessary):
  • SubmitMode.Auto http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders&realOData=true&updateGroupId=myAutoGroup
  • SubmitMode.Direct http://localhost:8080/testsuite/test-resources/sap/ui/core/demokit/sample/common/index.html?component=odata.v4.SalesOrders&realOData=true&updateGroupId=myDirectGroup

REALIZE and CHECK

    Create a new Sales Order

  1. Press the Create button for the Sales Order List
  2. In the developer tools observe that the POST ($batch with POST inside) request fails because 'Note' property is initial
  3. Type in some value for the 'Note' property and observe that the POST request is restarted successfully with the current value

    4. Change the new created or an already existing Sales Order

  4. In the 'Note' field type in the value RAISE_ERROR
  5. You should get an error that this value is not allowed
  6. Correct the value and observe that a new PATCH request is automatically send when leaving the field.
